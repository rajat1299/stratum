01 / The Problem
Agents are amnesiacs.
Your agent crashes. The sandbox dies. The work vanishes. There's no undo, no history, no trail. Just a cold restart and a blank slate.
0

files survive a sandbox restart.

Ephemeral compute means ephemeral work. Every session starts cold. Context rebuilt from scratch. Every single time.

Buckets don't branch.

S3 stores bytes. It doesn't track who changed what, or when, or why. Two agents write the same file. One wins. You'll never know which.

Git wasn't built for this.

Rate limits. Clone latency. Large-file friction. Agents don't want to git push. They want to open a file and write.

The missing layer.

Intelligence. Orchestration. Isolation. The agent stack has everything except the one thing that makes work persist: storage that understands history.

02 / Core Architecture
Built for machine precision.
Branches. Diffs. Access control. Audit trails. The primitives that made code manageable, applied to every file your agent touches.

Zero-Copy Branching

Branch the filesystem in microseconds. Ten agents, ten hypotheses, zero interference. Merge what works. Discard what doesn't.

POSIX Compliance

No new APIs to learn. Stratum mounts natively. Your agents call open(), read(), write(). We handle versioning underneath.

Sparse Materialization

A million files in the repo. Your agent needs three. Stratum loads exactly those three. Petabyte-scale context, byte-level precision.

Cryptographic Provenance

Every edit signed to a specific agent. Every decision traceable. Built for industries where the audit trail isn't optional.

03 / Integration
Three lines to mount. Zero lines to learn.
FUSE daemon for native mounts. TypeScript and Python SDKs for everything else. Your agent writes files. Stratum writes history.

Native mount

FUSE daemon. One command. Every process in your sandbox reads and writes Stratum like local disk. No SDK, no config, no ceremony.

SDK mount

No FUSE? No problem. TypeScript and Python SDKs give you the same semantics in-process. Each mount is isolated. Each mount enforces its own permissions.

Built-in shell

Give your agent a bash environment backed by Stratum. Standard unix tools. Versioned storage. Zero configuration.

bash typescript python 
# 1. Mount Stratum via CLI
$ stratum mount --target /mnt/agent_workspace --repo corp/legal_ai

# 2. Agent Python Integration
from stratum_sdk import StratumFS

# Initialize filesystem connection
fs = StratumFS("/mnt/agent_workspace")

# Agent starts reasoning, branch the filesystem
branch_id = fs.checkout_new("hypothesis_a")

# Agent writes findings (captured as a commit)
with open("/mnt/agent_workspace/analysis.txt", "w") as f:
    f.write("Contract clause 4.2 violates...")

# If agent hallucinates, rollback instantly
fs.rollback(branch_id, steps=1)
